# 智谱清言多智能体内容生产线设计方案

## 目标
搭建一条可在智谱清言工作流中运行的内容生产线，覆盖：
热点选题 → 素材采集 → 大纲设计 → 初稿撰写 → 质量审核 → 优化定稿 → 多平台适配

本版本的核心要求是：
- 所有 Agent 之间不用 JSON 传递中间结果
- 所有中间结果都采用自然语言分段展示
- 用户在前台能看到每一步“输入了什么、处理了什么、交接了什么”
- 条件分支通过固定文字关键词触发，不依赖 JSON 字段

## 采用方案
采用“方案 B：评审闭环型”，共 8 个智能体：

1. 趋势猎手
2. 素材采集员
3. 大纲架构师
4. 标题爆破手
5. 网感主笔人
6. 质量巡检员
7. 金句提炼师
8. 多平台适配师

## 设计原则
- 主链路必须串行，保证内容上下文持续累积
- 质量巡检员拥有分支判断权
- 若审核不通过，则回退给大纲架构师或网感主笔人
- 最多返工 2 次，避免工作流无限循环
- 联网搜索仅开放给“趋势猎手”“素材采集员”“质量巡检员”
- 其余智能体默认关闭联网能力，减少噪声与幻觉风险
- 全流程采用“文字交接单”形式输出，便于前台直接展示

## 输入参数
建议工作流入口统一收集以下变量：
- topic_direction：用户主题方向，例如“AI智能体”
- target_platforms：目标平台，例如“微信公众号、小红书、知乎、抖音”
- audience：目标受众，例如“AI从业者、产品经理、普通职场人”
- style_pref：风格偏好，例如“专业但有网感”
- length_pref：期望篇幅，例如“1200-1800字”
- score_threshold：审核阈值，默认 7
- max_rework_rounds：最大返工轮次，默认 2
- freshness_window：热点时间窗口，例如“近7天/近30天”
- must_include：必带信息点
- must_avoid：禁忌表达、敏感词、夸大承诺

## 输出产物
主产物建议至少包含：
- 热点候选清单
- 素材整理纪要
- 内容大纲
- 标题方案
- 初稿正文
- 审核评分与返工建议
- 定稿正文
- 金句清单
- 多平台改写版本
- 配图提示词（可选扩展）

## 联网与能力开关
- 趋势猎手：联网开启
- 素材采集员：联网开启
- 大纲架构师：联网关闭
- 标题爆破手：联网关闭
- 网感主笔人：联网关闭
- 质量巡检员：联网开启
- 金句提炼师：联网关闭
- 多平台适配师：联网关闭

## 统一交接格式
所有 Agent 必须使用统一的自然语言交接格式，禁止使用 JSON、表格、代码块作为中间传递格式。

建议每个 Agent 都输出以下 5 个部分：

- 【本轮任务】
- 【输入信息理解】
- 【处理结果】
- 【给下一位Agent的交接】
- 【风险与注意事项】

其中“处理结果”必须写清楚本节点真正做了什么，不能只写结论。

## 审核逻辑
质量巡检员从四个维度评分：
- 信息准确度
- 网感强度
- 逻辑连贯性
- 平台匹配度

评分建议使用文字格式直接展示，例如：
- 信息准确度：8/10
- 网感强度：7/10
- 逻辑连贯性：8/10
- 平台匹配度：6/10
- 综合得分：7.25/10

质量巡检员必须额外输出以下固定字段，便于条件分支识别：
- 审核结论：通过 / 不通过
- 建议回退对象：大纲架构师 / 网感主笔人 / 无
- 当前返工轮次：第 X 轮，共 2 轮
- 是否触发返工：是 / 否

## 分支规则
条件节点建议匹配固定文本关键词：

### 通过分支
当质量巡检员输出中包含：
- 审核结论：通过

则进入：
- 金句提炼师 → 多平台适配师 → 结束

### 返工到大纲架构师
当输出中同时包含：
- 审核结论：不通过
- 建议回退对象：大纲架构师
并且返工次数未达到上限

则进入：
- 大纲架构师 → 网感主笔人 → 质量巡检员

### 返工到网感主笔人
当输出中同时包含：
- 审核结论：不通过
- 建议回退对象：网感主笔人
并且返工次数未达到上限

则进入：
- 网感主笔人 → 质量巡检员

### 超过返工上限
当审核不通过且返工次数达到 2 次时：
- 输出当前最优版
- 明确提示“建议人工复核”
- 结束流程

## 建议节点形态
智谱清言工作流中建议使用如下节点类型：
- 开始节点
- 大模型节点（各智能体）
- 条件判断节点（根据固定文本判断）
- 变量设置节点（记录返工轮次）
- 结束节点

## 建议变量
为了适配“文字传递”，建议将每一步的输出都保存为完整文本变量：

- trend_handoff_text
- material_handoff_text
- outline_handoff_text
- title_handoff_text
- draft_handoff_text
- review_handoff_text
- quote_handoff_text
- platform_handoff_text
- rework_round

说明：
- 前 8 个变量都保存整段自然语言输出
- 下游 Agent 直接读取上游完整文本并继续处理
- 不要求拆成字段化 JSON

## 推荐运行顺序
开始
→ 趋势猎手
→ 素材采集员
→ 大纲架构师
→ 标题爆破手
→ 网感主笔人
→ 质量巡检员
→ 条件分支
  → 通过：金句提炼师 → 多平台适配师 → 结束
  → 不通过且返工次数未达上限：
       若结构问题：大纲架构师 → 网感主笔人 → 质量巡检员
       若表达问题：网感主笔人 → 质量巡检员
  → 不通过且返工次数达到上限：
       输出当前最优版 + 人工复核提示 → 结束

## 落地建议
如果智谱清言工作流后台需要变量传递，请把每个 Agent 的整段输出保存为“长文本变量”，不要要求 Agent 产出 JSON。
真正需要自动判断的部分，只保留少量固定关键词，例如：
- 审核结论：通过
- 审核结论：不通过
- 建议回退对象：大纲架构师
- 建议回退对象：网感主笔人
